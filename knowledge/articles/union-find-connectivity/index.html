01

DYNAMIC CONNECTIVITY

两种操作描述持续变化的分组

find(x)

找到元素所属集合的代表根。

representative
union(a, b)

把两个集合合并为一个集合。

merge
connected(a, b)

比较两者代表根是否相同。

same set

模型适合连接只增加、不删除的场景。若需要频繁删边或查询具体路径，应选择动态图、搜索或其他专用结构。

02

PARENT FOREST

每个集合是一棵指向根的树

parent 数组记录每个节点的父节点，根节点指向自己。find 沿父指针上行直到根，union 则让一棵树的根指向另一棵树的根。

核心模型PSEUDOCODE
find(x):
  while x != parent[x]: x = parent[x]
  return x

union(a, b):
  rootA = find(a); rootB = find(b)
  if rootA != rootB: parent[rootA] = rootB

直接随意挂接可能形成很高的树，使 find 退化。优化目标就是控制树高。

03

WEIGHTED UNION

把小树挂到大树下面

按大小或秩合并时，较小树的根指向较大树的根，避免大树反复被整体加深。规模数组只在根节点处有意义，合并后更新新根的规模。

  • ✦先比较根，不比较原始节点
  • ✦同一集合重复 union 不应修改规模
  • ✦组件数量可在真正合并时减一
  • ✦索引必须经过范围校验
04

PATH COMPRESSION

查询顺便把路径压平

路径压缩让 find 访问过的节点直接或逐步指向更高祖先。与按规模合并组合后，一系列操作的摊还成本非常接近常数，严格界通常以反 Ackermann 函数表达。

摊还结论针对操作序列

不能把它解释为每一次 find 都严格 O(1)；初始化仍需要 O(n) 空间和时间。

05

APPLICATIONS

哪些问题适合并查集

  • 01账户、设备或节点的去重归并
  • 02无向图是否形成环
  • 03Kruskal 最小生成树中的边筛选
  • 04网格岛屿随连接加入后的组件数量
  • 05离线批量处理等价关系

业务实体通常不是连续整数，需要先建立 ID 到内部索引的稳定映射，并控制映射的内存和生命周期。

06

BOUNDARIES

并查集不能回答什么

  • ✦不能直接给出两点之间的实际路径
  • ✦不能高效支持任意删除连接
  • ✦不能表达有向可达性和边权最短路
  • ✦普通实现不是线程安全结构
  • ✦代表根是内部结果，不应直接充当长期业务主键

智能卷轴 · Vector Retrieval
01

VECTOR REPRESENTATION

Embedding 把输入映射为固定维度的稠密向量

语义相近的句子通常在向量空间中更接近，可用于检索、聚类、去重和推荐。查询与文档必须使用兼容模型和相同预处理；模型升级会改变空间，不能把新旧向量无条件混用。

  • ✦记录 embeddingModel、维度和归一化方式
  • ✦保留原文、文档 ID、片段 ID 和版本
  • ✦批量生成控制请求大小、速率和失败重试
  • ✦输入先做权限分区和敏感数据治理
  • ✦模型升级采用新索引或明确重建流程
  • ✦删除原文时同步删除向量与缓存
02

SIMILARITY

距离函数必须与模型和索引配置一致

Cosine

关注向量夹角，常用于归一化语义向量。

1 - cos θ
Inner Product

使用点积，向量范数会影响得分。

x · y
Euclidean

计算几何距离，对尺度敏感。

||x-y||₂
Threshold

阈值应由标注数据校准，不跨模型直接复用。

score

Top-K 只保证返回排名前 K 的候选，即使所有候选都不相关。生产检索通常还需要最低质量阈值、无结果路径和下游重排。

03

CHUNKING

分块要保留语义单元和来源结构

  1. 01按标题、段落、代码块或表格等自然边界切分
  2. 02片段过长会混入多个主题并消耗上下文
  3. 03片段过短会丢失解释条件和实体关系
  4. 04适量重叠用于保留跨边界信息，避免大量重复
  5. 05为片段附加章节路径、页码、时间和版本
  6. 06查询结果可向父章节扩展，而非盲目扩大块

分块大小不是通用常数。接口文档、法律条款、FAQ 和源代码的语义结构不同，应分别用真实查询评测。

04

VECTOR INDEX

近似索引用召回率换取延迟和容量

精确搜索遍历全部向量，结果可预测但随规模增长变慢。HNSW、IVF 等近似最近邻索引通过图或分区减少候选，构建时间、内存、查询延迟与召回率之间需要权衡。

  • ✦以业务 Top-K 召回率和端到端延迟评估
  • ✦索引参数变更使用同一标注集比较
  • ✦监控索引大小、构建时间和查询尾延迟
  • ✦更新与删除的可见时间写入数据契约
  • ✦备份同时覆盖元数据、向量和模型版本
  • ✦容量估算包含向量、图结构和副本开销
05

HYBRID & FILTER

语义检索、关键词和元数据过滤互相补足

精确编号、专有名词和错误码适合关键词检索，语义问法适合向量检索。混合检索可分别召回再融合；租户、文档状态、语言和时间等条件应在候选阶段尽早过滤。

过滤不是授权的替代品

服务端必须按当前主体和数据范围构造过滤条件，并在返回原文前再次校验资源权限。不能让模型自己决定可访问哪些文档。

06

QUALITY & OPERATIONS

用标注查询衡量召回，而不是凭展示效果判断

  1. 01收集真实问题、正确文档与困难负例
  2. 02计算 Recall@K、MRR 或与业务匹配的排序指标
  3. 03分别比较分块、模型、索引、过滤和融合策略
  4. 04记录无结果、低分、权限过滤和过期文档比例
  5. 05监控嵌入失败、索引延迟和数据版本漂移
  6. 06重建索引时采用版本切换并验证完整性

智能卷轴 · Agent Runtime
01

OBSERVE · DECIDE · ACT

每轮循环都有明确输入、动作和停止条件

最小智能体循环STATE MACHINE
读取当前任务与状态
  → 组装受控上下文
  → 模型提出回答 / 工具调用 / 澄清 / 结束
  → Runtime 校验并执行
  → 保存观察结果和状态
  → 判断完成、暂停、失败或下一轮

停止条件包括任务完成、用户取消、需要确认、达到轮次/时间/成本预算、不可恢复失败。不能只依靠模型自己说“完成”。

02

EXPLICIT STATE

把任务状态存成结构，而不是藏在长对话里

  • ✦taskId、owner、tenant 和创建时间
  • ✦目标、成功条件和当前状态枚举
  • ✦已确认事实、待办步骤和未解决问题
  • ✦允许工具、授权范围和确认记录
  • ✦轮次、Token、时间和费用预算
  • ✦模型、Prompt、工具和数据版本

状态转移由 Runtime 校验，例如 WAITING_APPROVAL → RUNNING 需要有效用户确认。模型输出只是触发转移的候选事件，不直接修改持久状态。

03

MEMORY

记忆按用途、生命周期和可信度分层

工作记忆

当前任务需要的状态与证据，生命周期最短。

working
会话摘要

压缩历史决策和未完成事项，必须可回溯原事件。

summary
长期偏好

经用户确认、可查看和删除的稳定偏好。

profile
外部知识

按权限检索的文档与事实，不等同于用户记忆。

knowledge

自动写入长期记忆会制造隐私、错误固化和跨任务污染。敏感信息、临时指令和未经确认的模型推断不应作为长期事实保存。

04

CHECKPOINT & RESUME

在副作用边界保存可恢复检查点

  1. 01为每次运行和步骤分配稳定标识
  2. 02执行工具前保存意图、参数摘要和幂等键
  3. 03执行后保存结果、资源版本和下一状态
  4. 04恢复时先查询外部动作是否已完成
  5. 05重复消息按步骤标识去重并重放既有结果
  6. 06状态 Schema 变更提供明确迁移和版本检查
检查点不能让不可幂等动作自动安全

若外部系统不支持幂等或查询状态，恢复前必须人工确认，不能因本地缺少结果就再次执行。

05

BUDGET & CONCURRENCY

预算和并发控制防止智能体无限消耗与互相覆盖

  • ✦限制最大轮次、墙钟时间、Token、工具次数和费用
  • ✦对循环重复动作和无进展状态及时终止
  • ✦相同任务使用租约或乐观版本控制并发运行
  • ✦并行子任务有最大宽度和统一取消
  • ✦工具 Deadline 不超过任务剩余预算
  • ✦高风险动作不可因预算将尽而跳过确认
06

OPERATIONS

可观测智能体记录决策结果，而非泄露隐藏推理

  1. 01Trace 覆盖模型调用、工具调用、检索、确认和状态转移
  2. 02记录可审计的决策摘要、输入来源和结果，不记录隐藏思维链
  3. 03监控成功率、轮次、工具错误、暂停、成本和人工接管
  4. 04为卡死、重复调用、上下文超限和检查点失败告警
  5. 05生产轨迹脱敏后进入离线评测和失败分类
  6. 06模型、Prompt 或工具变化均进行轨迹回归

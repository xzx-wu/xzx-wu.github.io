02

RESIZABLE ARRAY

ArrayList 是通用默认选择

ArrayList 使用可扩容数组。规范保证 get、set、size 等常数时间，追加是摊还常数时间；在中间插入或删除需要移动后续元素。连续引用带来较好的遍历局部性。

  • ✦已知大致规模时设置初始容量，减少重复扩容
  • ✦批量删除优先使用迭代器或 removeIf，避免下标错位
  • ✦subList 返回视图，不是独立副本
  • ✦fail-fast 只能帮助发现错误，不能作为并发正确性机制
03

LINKED NODES

LinkedList 的 O(1) 有前提

LinkedList 是双向链表并实现 List 与 Deque。只有已经位于目标节点，或在两端操作时，链接修改才是常数成本；按下标定位仍需从头或尾遍历。节点对象、前后指针和较差的缓存局部性也会增加实际成本。

“中间插入 O(1)”常被误用

如果业务只有下标，先定位节点是 O(n)。因此频繁按索引访问时，LinkedList 往往不是合适选择。

04

ARRAY DEQUE

栈与队列优先考虑 ArrayDeque

ArrayDeque 是可扩容数组实现的双端队列，不允许 null。官方文档指出，它作为栈通常快于 Stack，作为队列通常快于 LinkedList。它适合 BFS 队列、单调队列、撤销栈和两端滑动窗口。

统一的双端语义JAVA API
Deque<Task> deque = new ArrayDeque<>();
deque.addLast(task);           // 队尾入队
Task next = deque.pollFirst(); // 队头出队，空队列返回 null

deque.push(task);              // 栈顶压入
Task top = deque.pop();        // 栈顶弹出，空栈抛异常
05

API SEMANTICS

失败语义也属于设计

add / remove / element

失败时抛异常，适合违反约束即为程序错误。

exception
offer / poll / peek

失败返回特殊值，适合正常处理空队列或容量不足。

special value

由于 ArrayDeque 禁止 null，poll 和 peek 返回 null 可以明确表示没有元素。若业务值本身允许 null，应在边界层消除歧义。

06

DECISION

工程选型清单

  • 01随机访问和顺序遍历：优先数组或 ArrayList
  • 02栈、队列与双端操作：优先 ArrayDeque
  • 03确实持有节点位置并频繁拼接：再考虑链式结构
  • 04多线程共享修改：选择明确的并发容器或外部同步
  • 05用真实规模测量遍历、分配与 GC，而非只背复杂度

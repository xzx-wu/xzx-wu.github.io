JVM 档案 · Native Diagnostic Tools
01

EVIDENCE SAFETY

采集前先评估成本

  • ✦记录 PID、主类、容器、时间、JDK 版本和故障现象
  • ✦命令通常需要在同一主机、以相同有效用户和组运行
  • ✦容器内外 PID 可能不同，应在目标 PID 命名空间确认
  • ✦线程栈、直方图、堆转储的影响逐步增加
  • ✦控制采集次数和文件路径，避免故障时把磁盘写满

Oracle 的 JDK 21 故障排查文档建议优先使用较新的 jcmd，而不是首先依赖早期的 jstack、jinfo 和 jmap 工具。

02

JCMD

先查看当前 JVM 支持的诊断命令

低风险事实采集SHELL
jcmd -l
jcmd 2468 help
jcmd 2468 VM.version
jcmd 2468 VM.command_line
jcmd 2468 VM.flags
jcmd 2468 VM.system_properties
jcmd 2468 VM.uptime

每个诊断命令都有独立影响级别和参数。使用 jcmd 2468 help CommandName 查看当前 JVM 的实际说明，不照搬其他版本命令。

常见诊断请求SHELL
jcmd 2468 Thread.print -l
jcmd 2468 GC.class_histogram
jcmd 2468 VM.native_memory summary
jcmd 2468 GC.heap_dump /tmp/app-2468.hprof

Native Memory Tracking 必须在 JVM 启动时启用，未开启时 VM.native_memory 无法补采历史。堆转储文件可能接近堆大小，需要先检查磁盘和权限。

03

THREAD DUMP

连续线程栈比单次快照更可靠

jcmd 优先，jstack 作为补充SHELL
jcmd 2468 Thread.print -l > /tmp/thread-01.txt
sleep 5
jcmd 2468 Thread.print -l > /tmp/thread-02.txt

jstack -l 2468 > /tmp/jstack.txt
  • 01高 CPU：找连续多次出现在相同业务栈的 RUNNABLE 线程
  • 02锁竞争：查看 BLOCKED 线程、持锁对象和拥有者
  • 03线程池耗尽：关注工作线程是否都卡在同一外部调用
  • 04死锁：查看工具报告和互相等待的锁依赖

线程栈中的 native、WAITING 或 parked 并不天然异常，要结合线程池职责、请求数量和持续时间判断。

04

HEAP EVIDENCE

先直方图，再决定是否转储

Class Histogram

按类查看实例数与浅大小，适合快速寻找异常增长。

GC.class_histogram
Heap Dump

保存完整堆供 MAT/JMC 等离线分析，文件大且影响更高。

GC.heap_dump
jmap

早期工具仍可使用，但应优先确认 jcmd 是否已提供所需能力。

jmap -histo
不要反复执行 live 直方图

jmap -histo:live 等“只统计存活对象”的操作可能触发垃圾收集。高负载实例上应先评估暂停风险。

有界堆采集SHELL
df -h /tmp
jcmd 2468 GC.class_histogram > /tmp/histo.txt
jcmd 2468 GC.heap_dump /tmp/app-$(date +%s).hprof
05

JSTAT

观察一段时间内的 GC 变化

每秒一次，共十次SHELL
jstat -gcutil 2468 1000 10
jstat -class 2468 1000 10
  • ✦比较采样间隔内年轻代和老年代使用率变化
  • ✦观察 YGC、FGC 次数及累计时间是否快速增长
  • ✦类加载和卸载持续增长可辅助判断动态加载问题
  • ✦不同收集器、JDK 版本的列含义可能不同，先看当前帮助

jstat 是计数器视角，无法直接解释对象为什么存活、线程为什么阻塞。需要和 GC 日志、JFR、线程栈或堆分析结合。

06

SCENARIO FLOW

按现象选择证据

高 CPU

系统进程确认 → 连续线程栈 → JFR/async-profiler。

thread evidence
响应卡死

线程栈 → 锁与线程池 → 外部依赖时间线。

blocking
内存上涨

GC 采样 → 类直方图对比 → 必要时 heap dump。

retention
Native 内存

RSS 与堆对比 → NMT → 线程栈、直接内存、库与映射。

off-heap

采集文件要包含时间戳、实例标识和命令说明，并在离线分析后按安全规范清理。

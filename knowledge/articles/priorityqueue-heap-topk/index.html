01

HEAP INVARIANT

完全二叉树压入连续数组

二叉堆通常以数组保存完全二叉树。最小堆要求父节点不大于子节点，最大堆则相反。它只维护局部父子关系，不维护兄弟或跨子树的全序。

零基数组下标关系FORMULA
parent(i) = (i - 1) / 2
left(i)   = 2 * i + 1
right(i)  = 2 * i + 2

03

JAVA PRIORITY QUEUE

PriorityQueue 默认是最小堆语义

PriorityQueue 按自然顺序或 Comparator 决定堆顶，不允许 null，也不是线程安全容器。API 保证 offer/poll/add/remove(Object) 等操作的成本层级，并明确 iterator 不保证特定顺序。

  • ✦需要完整排序输出时持续 poll，或复制后另行排序
  • ✦比较器使用 comparing、comparingInt 等方法，避免减法溢出
  • ✦入队后不要修改影响优先级的字段
  • ✦并发访问使用专门的并发优先队列或外部同步
04

BOUNDED TOP K

保留 K 个候选，而不是排序全部 N 个元素

寻找最大的 K 个元素时，维护容量为 K 的最小堆：堆未满则加入；堆满后仅当新值大于堆顶时替换。扫描结束后堆中即为候选集。

流式 TopKPSEUDOCODE
for each value in stream:
  if heap.size < K: offer(value)
  else if value > heap.peek: poll(); offer(value)

time  = O(N log K)
space = O(K)

如果 K 接近 N，完整排序或选择算法可能更合适；若数据分布在多节点，可先计算分区 TopK，再合并候选。

05

PITFALLS

优先队列的四个常见误区

  1. 01把 iterator 输出当作排序结果
  2. 02修改已入队对象的排序字段却不重新入队
  3. 03为每条数据重新排序整个列表
  4. 04忽略相同优先级下的稳定顺序需求
PriorityQueue 不保证稳定性

若相同优先级必须按到达顺序处理，应把单调序号纳入比较键，并处理序号溢出和持久化边界。

06

DECISION

什么时候使用堆

  • ✦反复获取当前最小值或最大值
  • ✦只关心 TopK，而非所有元素完整排序
  • ✦调度任务按优先级消费
  • ✦合并多个有序流时保存当前候选
  • ✦需要任意元素快速定位删除时，单独的堆可能不够

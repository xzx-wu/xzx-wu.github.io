智能卷轴 · Model Context Protocol
01

HOST · CLIENT · SERVER

Host 管理用户体验，Client 维护一条 Server 连接

Host

AI 应用本体，负责模型、权限、同意、上下文和整体策略。

application
Client

Host 内的协议组件，与一个 MCP Server 建立会话。

connection
Server

暴露 Resources、Prompts、Tools 等能力。

capability
Base Protocol

使用 JSON-RPC 2.0 请求、响应和通知。

JSON-RPC

一个 Host 可以连接多个 Server，并为每条连接隔离凭据、能力和数据。Server 的工具说明和返回内容都应视为不可信输入，Host 必须执行自己的安全策略。

02

LIFECYCLE

初始化阶段协商版本和双方能力

  1. 01Client 发送 initialize，声明协议版本、能力和实现信息
  2. 02Server 返回选定版本、自身能力和实现信息
  3. 03Client 发送 initialized 通知后进入正常操作
  4. 04双方只调用已经协商并实际支持的能力
  5. 05请求 ID 在会话内唯一，响应匹配原请求
  6. 06关闭连接时取消未完成工作并释放资源

客户端应明确发送支持版本，并处理不兼容版本错误。本文以 2025-11-25 稳定规范为基础，预览候选版本不能当作已发布兼容承诺。

03

PROTOCOL FEATURES

Resources、Prompts 与 Tools 表达不同控制关系

  • ✦Resources 提供文件、记录等上下文，由应用选择如何使用
  • ✦Prompts 提供可发现模板，通常由用户主动选择
  • ✦Tools 暴露可执行函数，由模型提出调用
  • ✦Client 可提供 Roots 限定服务器可见文件系统边界
  • ✦Sampling 和 Elicitation 等客户端能力需按协商和同意执行
  • ✦能力描述不是授权结果，实际调用仍需校验
04

TRANSPORT

本地进程与远程 HTTP 使用不同信任模型

stdio

Host 启动本地 Server，通过标准输入输出交换 JSON-RPC。

local process
Streamable HTTP

通过 HTTP POST 等机制承载远程消息和可选流式响应。

remote service
Protocol version

远程请求携带协商版本，服务端验证支持范围。

version
Session

会话标识需安全生成、绑定连接上下文并防止劫持。

session

stdio Server 的可执行文件、工作目录和环境变量都属于供应链与本机执行边界；远程 Server 则增加网络身份、OAuth、SSRF、重定向和跨域风险。

05

AUTHORIZATION & SAFETY

最小权限、用户同意和凭据隔离必须由 Host 落地

  • ✦安装或连接 Server 前展示来源、能力和数据范围
  • ✦高风险工具执行前展示实际参数与副作用并确认
  • ✦HTTP 授权使用适用的规范流程，不透传客户端 Token 给下游
  • ✦Server 只接受为自身签发且范围匹配的访问令牌
  • ✦限制 Server 可访问的文件、主机、网络和凭据
  • ✦防范 confused deputy、SSRF、会话劫持和恶意本地 Server
用户同意要绑定具体动作

“连接此 Server”不等于永久授权所有工具。工具列表变化、参数风险升高或访问新数据范围时应重新确认。

06

OPERATIONS

MCP 连接治理与排障清单

  1. 01记录协议版本、Server 版本、能力和 Schema 摘要
  2. 02监控连接、初始化、请求延迟、错误、取消和资源消耗
  3. 03限制消息大小、并发请求、超时和长任务数量
  4. 04日志脱敏凭据、资源正文和工具敏感参数
  5. 05升级 SDK 或规范前运行兼容与安全回归
  6. 06提供用户可见的断开、撤权和清理本地凭据入口

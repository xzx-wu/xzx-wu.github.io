02

REQUEST DTO

让约束贴近输入契约

CreateArticleRequest.javaJAVA
/** 创建文章的请求参数。 */
public class CreateArticleRequest {

    /** 文章标题。 */
    @NotBlank(message = "标题不能为空")
    @Size(max = 120, message = "标题不能超过 120 个字符")
    private String title;

    /** 文章正文。 */
    @NotBlank(message = "正文不能为空")
    private String content;

    // getter 与 setter 省略
}

输入 DTO 与持久化实体职责不同。不要为了复用而直接让实体承担外部输入约束，否则数据库字段、内部状态与 API 契约会被绑定在一起。

03

CONTROLLER

@Valid 与方法参数校验

控制器入口JAVA
/** 文章管理接口。 */
@RestController
@RequestMapping("/api/articles")
public class ArticleController {

    /** 创建一篇文章。 */
    @PostMapping
    public ArticleResponse create(
            @Valid @RequestBody CreateArticleRequest request) {
        return articleService.create(request);
    }

    /** 按编号查询文章。 */
    @GetMapping("/{id}")
    public ArticleResponse find(@PathVariable @Positive Long id) {
        return articleService.find(id);
    }
}

请求体对象校验失败通常进入 MethodArgumentNotValidException；方法参数约束可能由 Spring MVC 方法校验生成 HandlerMethodValidationException。处理器应覆盖项目实际使用的两类路径。

错误示例JSON

  • ✦稳定机器码用于前端分支，不依赖中文 message
  • ✦字段错误保留字段名和可展示信息
  • ✦traceId 用于关联服务端日志与链路
  • ✦HTTP 状态码表达协议层失败类型
05

CONTROLLER ADVICE

集中转换异常，不吞业务语义

统一处理器JAVA
/** 把参数校验异常转换为稳定错误响应。 */
@RestControllerAdvice
public class ApiExceptionHandler {

    /** 处理请求体字段校验失败。 */
    @ExceptionHandler(MethodArgumentNotValidException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public ApiError handleValidation(
            MethodArgumentNotValidException exception) {
        List<FieldErrorItem> errors = exception.getBindingResult()
            .getFieldErrors()
            .stream()
            .map(error -> new FieldErrorItem(
                error.getField(), error.getDefaultMessage()))
            .toList();
        return ApiError.validation(errors);
    }
}

不同异常要映射到明确的业务码和 HTTP 状态。未知异常记录完整堆栈后返回通用信息，不能把数据库 SQL、文件路径或内部类名暴露给客户端。

06

BOUNDARIES

校验不能替代后端权限

  • 01校验目标资源是否属于当前用户或组织
  • 02服务层再次确认状态流转和关键业务不变量
  • 03数据库唯一约束作为并发下的最终保证
  • 04日志中避免记录密码、Token 与完整敏感输入
  • 05批量错误数量设置上限，避免响应无限膨胀
不要把数据库异常原样返回

唯一键冲突应被转换为稳定业务错误，但转换前要判断具体约束，不能把所有 DataIntegrityViolationException 都误报成“已存在”。

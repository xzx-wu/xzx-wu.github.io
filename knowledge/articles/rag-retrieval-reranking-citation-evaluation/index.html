01

END-TO-END PIPELINE

RAG 是数据、检索与生成共同组成的系统

在线链路RAG
用户问题 + 身份上下文
  → 查询理解 / 改写
  → 权限过滤 + 多路召回
  → 重排 + 去重
  → 上下文组装 + 引用 ID
  → 受约束生成
  → 引用与输出验证

离线链路负责采集、解析、分块、元数据、Embedding 和索引。文档更新、删除与权限变化必须传播到索引，否则在线答案会引用已经失效的状态。

02

QUERY UNDERSTANDING

改写补全语义，但不能擅自改变用户目标

  • ✦从对话历史解析代词和省略条件
  • ✦识别精确编号、时间范围、语言与实体
  • ✦多问题拆分后分别检索，保留原始问题
  • ✦为关键词与向量检索生成不同查询表示
  • ✦不确定的关键条件应向用户澄清
  • ✦查询改写版本进入 Trace 但需脱敏
03

RETRIEVE & RERANK

召回追求不漏，重排负责把好证据放在前面

Sparse retrieval

擅长精确词、编号和罕见实体。

BM25
Dense retrieval

擅长语义近似和不同表述。

embedding
Fusion

融合多路排名，减少单一检索偏差。

RRF
Reranker

对较小候选集做更精细的查询—片段相关性判断。

cross-encoder

Top-K、融合权重和重排阈值应由评测集确定。增加候选会提高成本，也可能把无关或恶意内容带入上下文。

04

CONTEXT & CITATION

上下文组装保留来源、顺序和冲突信息

  1. 01去除重复片段并限制单一文档占比
  2. 02按相关性、新鲜度和权威级别排序
  3. 03每段附稳定 citationId、标题、版本和位置
  4. 04明确要求答案只依据给定证据
  5. 05输出后验证引用 ID 存在且支持对应陈述
  6. 06来源冲突时展示差异，不让模型静默选择
引用存在不等于引用支持结论

需要同时评估 citation correctness：所引片段是否真的蕴含答案，而不是仅与问题主题相关。

05

SECURITY BOUNDARY

检索内容既是证据，也是潜在不可信输入

  • ✦索引按租户、ACL 和文档状态携带权限元数据
  • ✦查询过滤从受信身份构造，不采用模型生成权限
  • ✦读取原文前再次校验资源归属
  • ✦文档中的“忽略指令”等文本始终视为数据
  • ✦文档采集执行来源验证、格式限制和恶意内容检测
  • ✦模型不能因检索命中自动获得写工具权限
06

LAYERED EVALUATION

分开评估检索失败和生成失败

Retrieval

正确证据是否进入候选和最终上下文。

Recall@K / MRR
Groundedness

答案中的陈述能否由上下文支持。

faithfulness
Answer quality

答案是否正确、完整、相关且表达清晰。

task score
System

权限、延迟、Token、成本、拒答和稳定性。

operations

评测集要包含可回答、不可回答、过期、冲突、权限拒绝和 Prompt Injection 样本。上线时记录各阶段候选数量、得分和版本，才能定位退化发生在哪一层。

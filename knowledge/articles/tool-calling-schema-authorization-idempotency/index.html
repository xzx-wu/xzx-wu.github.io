智能卷轴 · Tool Execution
01

TOOL CONTRACT

工具名称、描述和 Schema 必须准确且最小

  • ✦一个工具只承载一个清晰业务动作
  • ✦描述写明使用条件、不适用场景和副作用
  • ✦参数声明类型、必填、枚举、范围和单位
  • ✦关闭不需要的附加字段，拒绝模糊参数
  • ✦返回值区分成功、业务拒绝和系统失败
  • ✦工具版本与调用记录可追溯

不要设计万能 execute(command) 或 callApi(url, body)。窄工具更容易授权、审计、验证和评测，也能降低 Prompt Injection 扩大为任意执行的风险。

02

SELECTION VS EXECUTION

模型生成调用提案，运行时独立校验并执行

安全执行链RUNTIME
模型返回 toolName + arguments
  → 工具白名单检查
  → JSON Schema 校验
  → 身份 / 权限 / 数据范围校验
  → 风险与确认策略
  → 幂等执行
  → 结构化结果返回模型

工具结果也属于外部输入，可能过期、超大或包含恶意文本。运行时应限制大小、类型和来源；模型再次读取结果时，不能把其中的自然语言当作更高优先级指令。

03

AUTHORIZATION

授权依据来自登录主体，不来自模型参数

  1. 01运行时从受信会话取得用户、租户和角色
  2. 02工具服务校验操作权限和资源所有权
  3. 03模型传入的 userId、tenantId 只可作为业务对象，不作授权凭据
  4. 04读、写、删除、导出使用不同权限和凭据
  5. 05服务端对分页、批量、文件和目标地址设上限
  6. 06审计记录主体、资源、参数摘要、结果和 traceId
工具返回的数据范围必须与用户本身一致

Agent 不能因“需要完成任务”而绕过原系统权限。跨租户、跨组织或管理员操作必须走同样的后端授权链。

04

IDEMPOTENCY & APPROVAL

可重试动作使用稳定幂等键，高风险动作要求明确确认

只读查询

仍需权限、范围、超时和返回大小限制。

read
可逆写入

使用幂等键、版本和条件更新，返回实际新状态。

write
高风险动作

支付、发送、删除、发布等在执行前展示对象和影响并确认。

approval
批量动作

限制数量，逐项返回结果，避免部分成功不可见。

batch

确认应绑定工具名、参数摘要、资源版本和有效期。用户确认 A 不能被复用为执行参数已变化的 B。

05

FAILURE & RECOVERY

把可重试失败与业务拒绝明确区分

  • ✦参数非法直接返回字段错误，不让模型盲猜重试
  • ✦权限拒绝不能通过换参数或换工具绕过
  • ✦超时前设置端到端 Deadline 并传递取消
  • ✦仅对幂等、暂时性失败执行有限退避重试
  • ✦部分成功返回每项状态和补偿边界
  • ✦外部副作用失败进入人工或受控恢复流程
06

OBSERVABILITY & EVALUATION

同时评估工具选择、参数和真实执行结果

  1. 01记录工具候选、最终选择、参数校验和拒绝原因
  2. 02监控调用率、成功率、延迟、重试、确认和取消
  3. 03评测不该调用、应调用、工具混淆和参数边界样本
  4. 04加入直接与间接 Prompt Injection 攻击样本
  5. 05生产日志脱敏 Token、正文和个人数据
  6. 06工具 Schema 或权限变化后重新运行完整回归

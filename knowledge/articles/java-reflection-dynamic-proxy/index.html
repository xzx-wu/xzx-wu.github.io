02

INSPECT & INVOKE

安全读取结构并调用

读取方法JAVA
Class<?> type = Class.forName("com.example.OrderService");
Method method = type.getMethod("findById", Long.class);

try {
    Object result = method.invoke(service, 1001L);
} catch (InvocationTargetException exception) {
    // 业务异常被包装在 InvocationTargetException 中
    Throwable target = exception.getTargetException();
    throw new IllegalStateException("调用目标方法失败", target);
}

getMethod 面向公开且可继承的方法，getDeclaredMethod 只查询当前类声明。模块系统与强封装会限制深度反射，不能假设 setAccessible(true) 永远成功。

03

DYNAMIC PROXY

JDK 代理基于接口

创建代理JAVA
OrderService proxy = (OrderService) Proxy.newProxyInstance(
    OrderService.class.getClassLoader(),
    new Class<?>[] { OrderService.class },
    new LoggingInvocationHandler(target)
);

代理类在运行时实现指定接口，所有接口方法调用进入同一个 InvocationHandler。因此 JDK 动态代理无法直接为一个没有接口的普通类生成子类代理。

04

INVOCATION HANDLER

统一处理横切行为

日志代理JAVA
/** 为目标服务记录调用耗时。 */
public final class LoggingInvocationHandler implements InvocationHandler {
    /** 被代理的真实对象。 */
    private final Object target;

    /** 创建调用处理器。 */
    public LoggingInvocationHandler(Object target) {
        this.target = target;
    }

    /** 执行代理方法并保留原始异常语义。 */
    @Override
    public Object invoke(Object proxy, Method method, Object[] args)
            throws Throwable {
        long start = System.nanoTime();
        try {
            return method.invoke(target, args);
        } catch (InvocationTargetException exception) {
            throw exception.getTargetException();
        } finally {
            long elapsed = System.nanoTime() - start;
            System.out.println(method.getName() + ": " + elapsed);
        }
    }
}

处理器还会接收到 equals、hashCode、toString 等调用，必须明确它们的语义；异常若被错误包装，也可能改变接口声明的异常行为。

BOUNDARIES

反射的限制与风险

  • ✦编译器无法验证字符串方法名与参数组合
  • ✦非法访问、模块边界和安全策略可能在运行时失败
  • ✦频繁查找成员会增加开销，应缓存已解析的元数据
  • ✦反射写私有字段会绕过类的不变量与封装意图
  • ✦AOT 或原生镜像场景可能要求显式登记反射信息
不要吞掉目标异常

Method.invoke 会把目标异常包进 InvocationTargetException。框架代码应解包并保留原始失败原因，而不是统一改成无上下文的 RuntimeException。

06

PRACTICE

工程使用原则

  • 01把反射集中在框架适配层，不扩散到核心业务
  • 02启动阶段解析并缓存 Method、Constructor 等元数据
  • 03优先公开 API，避免依赖第三方组件私有成员
  • 04为代理明确接口、异常、Object 方法和线程安全语义
  • 05性能热点中评估 MethodHandle 或直接代码生成

01

THREE CONCERNS

原子性、可见性与有序性

原子性

一个操作或一组操作不可被观察为执行到一半。count++ 是读、计算、写的复合操作。

read → add → write
可见性

线程 A 写入共享状态后，线程 B 是否能够看到该写入。

write → visible read
有序性

编译器和处理器可在不破坏单线程语义的前提下重排，但跨线程观察可能不同。

program order
数据竞争

两个线程访问同一变量、至少一个是写，并且不存在 happens-before 顺序。

conflicting access
存在竞争的计数器JAVA
public final class UnsafeCounter {
    /** 多线程执行自增时会发生丢失更新。 */
    private int value;

    public void increment() {
        value++;
    }
}
02

ORDERING RULES

happens-before 是什么

happens-before 描述两个动作之间的内存可见性与顺序保证。如果动作 A happens-before 动作 B，那么 A 的结果对 B 可见，并且 A 在内存模型意义上排在 B 之前。

  • ✦同一线程中，前面的动作 happens-before 后续动作
  • ✦监视器解锁 happens-before 后续对同一监视器加锁
  • ✦volatile 写 happens-before 后续对该变量的读
  • ✦调用 Thread.start happens-before 新线程中的动作
  • ✦线程中的动作 happens-before 其他线程成功 join 返回
  • ✦happens-before 具有传递性
它不是墙上时钟顺序

happens-before 是语言规范定义的可见性关系。两个动作实际时间上先后发生，但如果缺少同步关系，程序仍可能存在数据竞争。

03

MONITOR LOCK

synchronized 提供什么

synchronized 使用对象监视器实现互斥。进入同一监视器保护的临界区前需要获得锁，退出时释放锁；释放与后续获取之间建立 happens-before。

线程安全计数器JAVA
public final class SynchronizedCounter {
    /** 由当前实例监视器保护的计数值。 */
    private int value;

    /**
     * 在同一监视器内完成读取、计算和写入，避免丢失更新。
     */
    public synchronized void increment() {
        value++;
    }

    /** 返回与写入使用同一锁保护的最新值。 */
    public synchronized int get() {
        return value;
    }
}

锁保护的是一条不变量，而不仅是某个字段。涉及多个字段的一致性更新时，应让相关读写遵循同一把锁和同一套边界。

04

VOLATILE FIELD

volatile 适合什么

volatile 读写具有明确的可见性和顺序语义，适合一个线程发布简单状态、其他线程读取的场景。它不提供复合操作的互斥。

停止标记JAVA
public final class Worker implements Runnable {
    /** 跨线程发布停止信号。 */
    private volatile boolean running = true;

    /** 请求任务循环结束。 */
    public void stop() {
        running = false;
    }

    @Override
    public void run() {
        while (running) {
            doOneTask();
        }
    }
}
volatile 不能让 value++ 原子化

自增包含读取旧值、计算新值和写回三个步骤。需要原子计数时使用锁、AtomicInteger、LongAdder，具体选择取决于是否需要复合不变量和读取精度。

05

SAFE PUBLICATION

对象如何安全发布

构造完成的对象必须通过有同步保证的方式交给其他线程，才能避免其他线程观察到未完成初始化或过期状态。

  • ✦在静态初始化过程中发布
  • ✦通过 volatile 引用发布
  • ✦在锁保护范围内写入和读取引用
  • ✦放入线程安全容器
  • ✦构造期间不要让 this 逃逸
  • ✦优先使用不可变对象简化共享
06

CHOOSING A TOOL

不要只在两个关键字中选择

简单状态发布

一个写线程更新标志，其他线程读取，没有复合不变量。

volatile
复合状态一致性

多步操作必须一起完成，或多个字段共同构成不变量。

synchronized / Lock
单变量原子操作

计数、比较并交换等可使用原子类，但要明确竞争强度。

Atomic*
消息传递

能用队列传递数据时，通常比多个线程随意共享可变对象更清晰。

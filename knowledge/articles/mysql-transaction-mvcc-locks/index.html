01

TRANSACTION BOUNDARY

先明确事务边界

事务把一组操作组织成不可随意拆分的业务单元。自动提交开启时，每条语句通常独立提交；显式事务则从 START TRANSACTION 开始，到 COMMIT 或 ROLLBACK 结束。

转账事务示意SQL
START TRANSACTION;

UPDATE account SET balance = balance - 100 WHERE id = 1;
UPDATE account SET balance = balance + 100 WHERE id = 2;

COMMIT;
事务要尽量短

不要在数据库事务中执行远程调用、长时间计算或等待用户输入。事务越长，版本清理越慢，持锁时间越长，冲突和连接池占用也越明显。

02

ISOLATION LEVELS

四种隔离级别

隔离级别决定并发事务之间可以观察到什么。InnoDB 默认使用 REPEATABLE READ，但实际项目应根据一致性要求和冲突成本选择。

READ UNCOMMITTED

可能读到其他事务尚未提交的数据，隔离最弱，业务系统很少主动选择。

允许脏读
READ COMMITTED

每次一致性读都会建立新的可见性视图，同一事务两次查询结果可能不同。

避免脏读
REPEATABLE READ

事务中的一致性读通常复用首次读取建立的视图，保持重复读取的一致性。

InnoDB 默认
SERIALIZABLE

进一步限制并发访问，隔离更强，但等待和冲突成本也更高。

并发能力最低
设置会话隔离级别SQL
SET SESSION TRANSACTION ISOLATION LEVEL READ COMMITTED;

SELECT @@transaction_isolation;
03

MULTI-VERSION

MVCC 如何提供一致性读

InnoDB 会保存行的事务信息，并利用 undo 中的历史版本构造一致性读取需要的旧值。读取视图根据事务可见性规则判断当前版本是否可见，不可见时沿版本链查找合适版本。

  • ✦当前事务自己的修改对自己可见
  • ✦未提交事务的修改不会进入普通一致性读
  • ✦历史版本来自 undo 信息，不是完整复制整张表
  • ✦长事务会让旧版本保留更久，增加清理压力
MVCC 不等于完全无锁

普通 SELECT 多数情况下使用一致性非锁定读；更新、删除、插入以及显式锁定读仍然需要锁。

04

READ MODES

一致性读与当前读

普通查询通常读取符合可见性规则的版本；需要基于最新数据修改时，应使用锁定读并在事务内尽快完成后续更新。

两种读取方式SQL
-- 一致性非锁定读：读取视图允许看到的版本
SELECT balance FROM account WHERE id = 1;

-- 锁定读：读取当前记录并申请排他锁
SELECT balance FROM account WHERE id = 1 FOR UPDATE;

FOR UPDATE 不是“防止任何人读取”，而是为后续修改建立并发控制。它必须放在明确的事务边界内，否则自动提交会让锁很快释放，无法保护后续业务步骤。

05

LOCK TYPES

记录锁、间隙锁与临键锁

记录锁

锁定索引中的具体记录。InnoDB 所谓行锁，本质上与扫描到的索引记录相关。

record lock
间隙锁

锁定两个索引记录之间的区间，主要限制其他事务在间隙内插入。

gap lock
临键锁

记录锁与其前方间隙锁的组合，常用于范围扫描并避免幻行。

next-key lock
意向锁

表级标记，说明事务准备在更细粒度上申请共享锁或排他锁。

IS / IX

锁范围取决于隔离级别、访问语句、索引和扫描范围。缺少合适索引时，语句可能扫描并锁定更多索引记录，因此“优化索引”也常常是减少锁冲突的重要手段。

06

DEADLOCK

死锁为什么会发生

两个事务以不同顺序持有资源并等待对方释放时，就可能形成循环等待。InnoDB 可以检测死锁并回滚其中一个事务，但应用必须正确处理失败并决定是否重试。

查看最近一次死锁SQL
SHOW ENGINE INNODB STATUS;
  • 01不同业务路径按相同顺序访问表和记录
  • 02事务保持短小，避免无关操作延长持锁时间
  • 03为过滤条件建立合适索引，缩小扫描与锁范围
  • 04捕获死锁异常，只对幂等且可安全重试的事务重试
07

PRACTICE CHECKLIST

事务实践清单

  • ✦明确业务原子性与事务开始、结束位置
  • ✦确认查询使用一致性读还是锁定读
  • ✦避免在事务里调用不受控的外部服务
  • ✦监控长事务、锁等待、死锁与连接占用
  • ✦更新语句必须使用有效过滤条件和索引
  • ✦重试策略要满足幂等性并设置明确上限

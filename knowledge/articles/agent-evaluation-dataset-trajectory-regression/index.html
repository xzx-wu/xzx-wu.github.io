智能卷轴 · Evaluation
01

EVALUATION TARGET

从用户任务和风险边界定义成功

  • ✦任务完成：结果是否满足业务成功条件
  • ✦事实与引用：结论是否被可信证据支持
  • ✦工具行为：是否选对工具和参数，副作用是否正确
  • ✦安全边界：是否拒绝越权、注入和敏感数据请求
  • ✦效率：轮次、延迟、Token、调用和费用
  • ✦体验：澄清是否必要，失败说明是否可行动

不同任务权重不同。客服答案、代码修复和生产变更不能共用一个总分掩盖关键失败，安全或权限红线通常应作为硬门槛。

02

EVALUATION DATASET

数据集覆盖典型任务、边界和真实失败

Golden

稳定核心用例和明确期望，用于版本间基线比较。

regression
Edge

缺失输入、冲突证据、长上下文和工具失败。

boundary
Adversarial

直接/间接注入、越权、数据外泄和高风险动作。

security
Production

经脱敏、去重和审核的真实失败与新分布。

feedback

训练、调参和最终评测数据应分开，避免反复针对测试集优化。每条样本记录来源、任务类型、期望、允许工具、权限上下文和版本。

03

LAYERED METRICS

结果、轨迹和系统指标分别定位问题

  1. 01结果层：正确性、完整性、格式、引用和拒答
  2. 02检索层：Recall@K、重排、权限过滤和文档新鲜度
  3. 03工具层：选择、参数、顺序、幂等、确认和最终状态
  4. 04轨迹层：无效步骤、重复循环、恢复和人工接管
  5. 05系统层：成功率、延迟分位、Token、费用和错误
  6. 06安全层：越权执行、数据泄露和注入攻击成功率

轨迹评测关注可观察动作和状态，不要求保存或评判模型隐藏思维链。这样既便于审计，也减少敏感内部推理泄露。

04

GRADERS

确定性检查优先，模型判分需要校准

  • ✦Schema、枚举、引用 ID 和工具参数使用代码检查
  • ✦精确答案使用单元测试或业务规则判定
  • ✦开放文本用明确 Rubric 分解多个维度
  • ✦模型判分器与人工标注样本定期比较
  • ✦随机输出多次运行并报告均值、分布和样本量
  • ✦关键失败由人工复核，不用单一总分覆盖
评测指标必须服务决策

上线门槛、灰度比较和故障回归的目标不同，应提前说明指标如何影响发布，而不是测完再挑一个好看的数。

05

ONLINE OBSERVATION

线上观测发现离线数据未覆盖的新分布

  • ✦Trace 关联模型、检索、工具、状态转移和用户结果
  • ✦监控成功、拒绝、人工接管、取消和重复循环
  • ✦收集显式反馈，同时避免把点赞直接等同正确
  • ✦抽样复核高风险、低置信和高成本轨迹
  • ✦日志与评测样本执行脱敏、访问控制和保留期限
  • ✦异常样本分类后回流离线集并记录修复版本
06

CONTINUOUS REGRESSION

模型、Prompt、工具和知识变化都要独立比较

发布前评测链CI / CANARY
固定代码与数据 → 比较模型 / Prompt 变化
固定模型与 Prompt → 比较检索 / 工具变化
全链路候选版本 → 运行黄金集 + 边界集 + 安全集
通过门槛 → 小流量灰度 → 线上指标与基线比较

保存每次运行的配置、数据集版本、随机参数和原始判定，才能重现差异。失败先按模型、检索、工具、权限、数据或评测器分类，再决定修复位置。

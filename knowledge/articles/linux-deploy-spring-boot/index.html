工程档案 · Linux 基础部署
01

PREPARATION

部署前准备

本文以使用 Yum 的 Linux 发行版为例，部署一个可执行的 Spring Boot JAR。开始之前，需要准备以下内容：

  • ✦可以登录的 Linux 服务器
  • ✦与项目匹配的 Java 版本
  • ✦可正常打包的 Spring Boot 项目
  • ✦服务器目录的写入权限
版本提醒

示例沿用 Java 8 软件包名称。实际部署时必须选择与项目编译版本一致的 JDK，不要机械照搬版本号。

02

JAVA RUNTIME

检查并安装 JDK

先检查服务器是否已经安装 Java。能够输出版本信息，就不必重复安装。

检查环境BASH
java -version
which java

如果没有安装，可以先查询仓库中的 OpenJDK 软件包，再安装项目需要的版本：

安装 OpenJDKBASH
sudo yum list 'java-*-openjdk*'
sudo yum install -y java-1.8.0-openjdk.x86_64
java -version

通过 Yum 安装后，JDK 通常位于 /usr/lib/jvm。如果服务器使用 DNF，可以把上述命令中的 yum 替换为 dnf。

03

PACKAGE

在本地打包项目

在项目根目录执行 Maven 打包。建议先完成一次正常测试；明确需要跳过测试时，再增加对应参数。

Maven 打包BASH
mvn clean package

# 确认需要跳过测试时使用
mvn clean package -DskipTests

命令成功后，可执行 JAR 通常位于项目的 target 目录。建议先在本地运行一次，排除打包问题：

本地验证BASH
java -jar target/demo-web-1.0.0-SNAPSHOT.jar
04

TRANSFER

上传 JAR 到服务器

先在服务器上准备应用目录：

服务器执行BASH
sudo mkdir -p /usr/local/app
sudo chown -R "$USER":"$USER" /usr/local/app

然后在本地终端使用 SCP 上传，并统一保存为简洁的文件名：

本地执行BASH
scp target/demo-web-1.0.0-SNAPSHOT.jar \
  your-user@your-server:/usr/local/app/app.jar
也可以使用 rz

习惯图形化终端上传时，可安装 lrzsz 后使用 rz。SCP 更通用，也更适合写入自动化流程。

05

RUN SERVICE

启动并检查服务

第一次部署建议先以前台方式启动，这样能立即看到配置、端口或数据库连接错误：

前台启动BASH
cd /usr/local/app
java -jar app.jar

确认可以正常启动后，按 Ctrl + C 结束前台进程，再改为后台运行：

后台启动BASH
nohup java -Xms256m -Xmx512m \
  -jar /usr/local/app/app.jar \
  --spring.profiles.active=test \
  > /usr/local/app/application.log 2>&1 &

echo $! > /usr/local/app/application.pid

nohup 让进程不再依赖当前终端；输出重定向到指定日志后，可以持续跟踪启动过程：

查看状态BASH
tail -f /usr/local/app/application.log
ps -fp "$(cat /usr/local/app/application.pid)"
06

SERVICE SCRIPT

编写管理脚本

命令确认无误后，可以用一个小脚本统一管理启动、停止和状态查询。保存为 /usr/local/app/app.sh：

app.shBASH
#!/usr/bin/env bash

APP_HOME=/usr/local/app
JAR_FILE="$APP_HOME/app.jar"
LOG_FILE="$APP_HOME/application.log"
PID_FILE="$APP_HOME/application.pid"
JAVA_OPTS="-Xms256m -Xmx512m"
SPRING_OPTS="--spring.profiles.active=test"

is_running() {
  [ -f "$PID_FILE" ] && kill -0 "$(cat "$PID_FILE")" 2>/dev/null
}

start() {
  if is_running; then
    echo "Application is already running, PID=$(cat "$PID_FILE")"
    return 0
  fi

  if [ ! -f "$JAR_FILE" ]; then
    echo "JAR file not found: $JAR_FILE"
    return 1
  fi

  nohup java $JAVA_OPTS -jar "$JAR_FILE" $SPRING_OPTS \
    > "$LOG_FILE" 2>&1 &
  echo $! > "$PID_FILE"
  echo "Application started, PID=$(cat "$PID_FILE")"
}

stop() {
  if ! is_running; then
    echo "Application is not running"
    rm -f "$PID_FILE"
    return 0
  fi

  PID=$(cat "$PID_FILE")
  kill "$PID"
  echo "Stop signal sent to PID=$PID"
  rm -f "$PID_FILE"
}

status() {
  if is_running; then
    echo "Application is running, PID=$(cat "$PID_FILE")"
  else
    echo "Application is not running"
  fi
}

case "${1:-}" in
  start) start ;;
  stop) stop ;;
  restart) stop && start ;;
  status) status ;;
  *) echo "Usage: $0 {start|stop|restart|status}"; exit 1 ;;
esac

添加执行权限后即可使用：

脚本命令BASH
chmod u+x /usr/local/app/app.sh

/usr/local/app/app.sh start
/usr/local/app/app.sh status
/usr/local/app/app.sh stop
生产环境建议

这个脚本适合学习和基础部署。正式生产环境更适合交给 systemd、容器平台或进程管理工具托管。

07

TROUBLESHOOTING

常见问题检查

Java 版本不匹配

出现 class version 相关错误时，对比本地编译 JDK 与服务器运行 JDK。

java -version
端口没有监听

先查看应用日志，再确认配置端口是否已经被其他进程占用。

ss -lntp | grep 8080
外网无法访问

检查云服务器安全组、系统防火墙和应用监听地址。

curl http://127.0.0.1:8080
日志无法写入

确认运行用户对应用目录和日志文件拥有写入权限。

ls -ld /usr/local/app

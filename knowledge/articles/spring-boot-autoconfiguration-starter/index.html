01

CONCEPTS

Starter、自动配置与属性绑定

Starter

一组有明确用途的依赖集合，减少用户手工选择相互兼容依赖的成本。

dependency bundle
Auto-configuration

在满足类路径、Bean 或属性条件时提供默认 Bean。

conditional beans
Configuration Properties

把外部配置绑定为类型安全对象，作为自动配置的输入。

typed settings

@SpringBootApplication 包含自动配置能力的启用入口。Starter 本身不是运行时扫描器，也不等于自动配置类；成熟组件常把 autoconfigure 与 starter 拆成两个模块。

03

CONDITIONS

条件装配与回退机制

@ConditionalOnClass

仅当所需类存在于类路径时启用，避免可选依赖缺失。

classpath
@ConditionalOnMissingBean

用户没有提供同类 Bean 时才创建默认实现，是回退机制的核心。

back off
@ConditionalOnProperty

根据配置开关或属性值启用功能。

feature flag
@ConditionalOnWebApplication

根据应用类型区分 Web 与非 Web 场景。

application type
自动配置应晚于用户决策

好的自动配置提供合理默认值，但允许用户通过 Bean 或属性明确覆盖；不要使用无条件 Bean 抢占应用自己的配置。

04

CONDITION REPORT

自动配置没有生效时怎么查

启用条件报告SHELL
java -jar archive-app.jar --debug

# 或在配置中开启
debug: true

条件评估报告会列出匹配和未匹配的自动配置及原因。排查时依次确认依赖是否存在、配置类是否进入候选、属性名称是否正确、用户是否已经定义了导致回退的 Bean。

05

CUSTOM CONFIG

实现一个最小自动配置

ArchiveMessageAutoConfiguration.javaJAVA
@AutoConfiguration
@ConditionalOnClass(MessageClient.class)
@EnableConfigurationProperties(MessageProperties.class)
public class ArchiveMessageAutoConfiguration {

    /**
     * 当应用没有自定义消息服务时，提供默认实现。
     */
    @Bean
    @ConditionalOnMissingBean
    public MessageService messageService(
            MessageClient client,
            MessageProperties properties) {
        return new DefaultMessageService(client, properties.getPrefix());
    }
}
MessageProperties.javaJAVA
@ConfigurationProperties("archive.message")
public class MessageProperties {

    /** 消息内容使用的前缀。 */
    private String prefix = "[Archive]";

    public String getPrefix() {
        return prefix;
    }

    public void setPrefix(String prefix) {
        this.prefix = prefix;
    }
}

自动配置类本身不应成为组件扫描目标。让 Spring Boot 从导入清单加载它，可以避免应用扫描顺序和包结构造成隐式依赖。

07

TESTING

验证默认值与回退边界

自定义自动配置至少应验证四类场景：必需类存在时成功装配、依赖缺失时不装配、用户提供 Bean 时回退、属性开关关闭时不装配。Spring Boot 提供的 ApplicationContextRunner 适合用小型上下文测试条件组合。

测试意图JAVA
contextRunner
    .withUserConfiguration(UserMessageConfiguration.class)
    .run(context -> {
        assertThat(context).hasSingleBean(MessageService.class);
        assertThat(context.getBean(MessageService.class))
            .isSameAs(context.getBean("customMessageService"));
    });

示例强调“用户 Bean 优先”的断言。实际项目还应为属性绑定失败、缺失依赖与多个候选 Bean 编写边界测试。

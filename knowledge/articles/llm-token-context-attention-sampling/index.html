01

TOKENIZATION

文本先被切分为模型词表中的 Token

Tokenizer 将字符序列映射为整数 ID。一个 Token 可能是单字、词片段、标点或空白组合，不同模型的词表和切分规则不同，因此字符数不能直接等同于 Token 数。

  • ✦输入、输出和工具结果都会占用 Token 预算
  • ✦中英文、代码与特殊字符的切分密度不同
  • ✦同一文本换模型后应重新估算长度
  • ✦截断必须保留指令、问题和关键证据
  • ✦日志避免记录完整敏感 Prompt
  • ✦成本统计使用实际请求返回的用量信息
02

SELF-ATTENTION

Attention 让当前位置按相关性读取其他位置

Transformer 使用查询、键和值计算位置之间的注意力权重，多头机制可以并行学习不同关系。位置编码为序列加入顺序信息，前馈网络在每个位置进一步变换表示。

缩放点积注意力CONCEPT
scores = Q · Kᵀ / √d
weights = softmax(scores + mask)
output = weights · V

注意力权重不是事实证明或可直接等同于人类解释。模型能关联上下文，但不会因此自动获得外部事实、最新数据或业务权限。

03

CONTEXT WINDOW

上下文窗口是一次推理可处理的有限工作区

指令

系统约束、任务目标、输出协议和安全边界。

instructions
历史

当前对话中保留的用户与助手消息。

history
证据

检索文档、工具返回和业务状态。

evidence
输出

为本次生成预留的最大空间。

completion

窗口更大不等于所有位置被同等利用。长上下文会增加延迟与成本，也可能让重要证据被噪声稀释。工程上应选择相关片段、压缩历史，并保留原始证据的可追溯引用。

04

AUTOREGRESSIVE GENERATION

模型逐步预测并把输出加入下一步上下文

  1. 01把输入 Token 送入模型得到下一个 Token 的概率分布
  2. 02按解码规则选择一个候选 Token
  3. 03将该 Token 追加到当前序列
  4. 04重复计算直到停止标记或输出上限

流式输出只是把已生成片段更早交给用户，不改变逐 Token 生成本质。应用仍应在完整结构完成后验证 JSON、引用、权限和业务约束。

05

SAMPLING

生成参数调节候选分布，而不是控制事实正确率

Temperature

调整分布平滑程度；更高通常增加多样性，也增加波动。

temperature
Top-p

从累计概率达到阈值的候选集合中采样。

nucleus
Maximum output

限制生成长度，防止成本和延迟失控。

max tokens
Stop condition

在协议结束标记或工具调用完成后停止。

stop

需要稳定分类和抽取时，优先使用清晰协议、较少自由度、结构校验和评测数据；不能仅靠降低 Temperature 解决幻觉。

06

ENGINEERING BOUNDARIES

把模型当作概率组件，而不是可信执行环境

  1. 01记录模型版本、Prompt 版本、用量、延迟和结果状态
  2. 02对输出执行 Schema、枚举、范围和业务校验
  3. 03外部事实通过检索或工具获得，并保存来源
  4. 04授权、金额计算和状态变更由确定性代码负责
  5. 05为超时、限流和上游错误设计显式失败结果
  6. 06任何模型或参数变更都用固定数据集回归

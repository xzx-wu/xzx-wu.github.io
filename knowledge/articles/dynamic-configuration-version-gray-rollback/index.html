01

SOURCES & PRECEDENCE

先确定唯一来源与覆盖规则

Spring Boot 支持属性文件、环境变量、命令行等外部配置来源，并定义覆盖顺序。配置治理应记录最终值来自哪里，避免同名键在镜像、环境和配置中心多处存在却无法解释。

  • ✦默认值只表达安全且普适的运行基线
  • ✦环境差异由外部配置注入
  • ✦命名包含业务域、组件和单位
  • ✦类型、范围、必填项在启动或发布时校验
  • ✦未知键和拼写错误不能静默忽略
  • ✦凭据由 Secret 管理，不放普通 ConfigMap
02

IMMUTABLE VERSION

每次发布形成不可变配置快照

配置版本包含内容、Schema、目标范围、创建人、审批人、时间和变更原因。运行实例报告当前版本，使故障现场能够回答“哪一批实例在什么时间应用了哪份配置”。

配置发布元数据SNAPSHOT
version: cfg-20260717-0042
schemaVersion: 3
target: prod / payment-service / canary-a
checksum: sha256:...
owner: payment-platform
changeTicket: CHG-1842

回滚是重新发布一个历史快照或等价新版本，不应直接覆盖历史记录。这样才能保留完整审计链和实际生效顺序。

03

ATOMIC REFRESH

先校验完整快照，再一次性切换引用

  1. 01接收配置版本通知并下载完整快照
  2. 02校验签名、Schema、类型、范围和依赖关系
  3. 03构造新的不可变配置对象
  4. 04完成必要预热或连接探测
  5. 05原子替换当前版本引用
  6. 06记录成功、拒绝和应用耗时

不是所有参数都适合热更新。线程池结构、端口、类加载和部分连接初始化通常需要重建或重启；应为每个键声明刷新方式，而不是假设改值必然即时安全生效。

04

GRAY TARGETING

灰度目标必须稳定、可解释且可撤销

配置可按环境、集群、实例组、租户或稳定哈希分桶定向。目标条件属于发布契约，必须避免使用随请求变化或无法追踪的随机规则。

权限

编辑、审核、发布、回滚和查看敏感值分离授权。

RBAC
审计

记录前后差异、操作者、审批、目标和结果。

audit
灰度

从内部或低风险单元开始，逐级扩大目标。

cohort
隔离

测试配置不能通过通配范围进入生产。

scope
05

VALIDATE & ROLLBACK

发布前阻止非法值，发布后观察业务结果

  • ✦静态校验类型、范围、枚举、单位和依赖
  • ✦影子或预发环境验证关键用例
  • ✦灰度观察错误率、延迟、容量和业务指标
  • ✦配置与对应应用版本兼容关系可查询
  • ✦回滚动作需授权，且保留当次原因与结果
  • ✦数据格式变更准备前后向兼容与迁移方案
回滚不能撤销已经发生的业务副作用

若配置已触发数据写入、消息发送或外部调用，还需要业务补偿和数据修复，不能只恢复一个旧值。

06

OPERATIONS

配置平台运行治理清单

  1. 01监控发布延迟、应用成功率、版本分布和漂移
  2. 02告警长期未收敛实例和反复刷新失败
  3. 03限制单次变更规模、频率和高风险时段
  4. 04为配置中心不可用保留最近成功快照
  5. 05演练错误配置、控制面故障与大规模回滚
  6. 06定期清理废弃键并核对所有消费方

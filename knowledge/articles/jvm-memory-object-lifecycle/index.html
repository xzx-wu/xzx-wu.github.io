运行时档案 · JVM Memory
01

SPECIFICATION

先分清规范与实现

JVM 规范描述运行时数据区应具备的语义，但不规定所有区域的物理位置、连续性或垃圾收集算法。HotSpot 的 Metaspace、TLAB 和分代布局都是实现选择，不能反推为所有 JVM 的强制结构。

常见误区

“方法区等于永久代”并不准确。方法区是规范定义的逻辑区域；永久代是旧版 HotSpot 的实现，现代 HotSpot 使用本地内存中的 Metaspace 承载相关元数据。

02

RUNTIME AREAS

六类运行时数据区

程序计数器

每个线程私有，记录当前线程执行位置；规范允许 native 方法执行时取值未定义。

per thread
JVM 栈

每个线程私有，以栈帧支持方法调用；深度不足可抛出 StackOverflowError。

frames
本地方法栈

支持非 Java 语言实现的方法，具体组织方式由虚拟机决定。

native
堆

线程共享，保存对象与数组，是垃圾收集器主要管理区域。

shared heap
方法区

线程共享，保存类结构、方法和字段信息等；规范允许不实现垃圾收集。

class metadata
运行时常量池

每个类或接口对应一个运行时常量池，是方法区的一部分。

constants
03

STACK FRAME

一次方法调用里有什么

  • ✦局部变量数组：参数和方法内部局部变量
  • ✦操作数栈：字节码指令计算时的工作区
  • ✦动态链接：把符号引用解析到运行时目标
  • ✦正常或异常返回所需的信息

栈帧大小通常可以从方法字节码和类结构确定。递归过深、方法局部状态过大或线程栈设置过小，都可能导致栈溢出；大量线程则会共同消耗进程的本地内存。

04

ALLOCATION

对象如何进入堆

执行 new 后，虚拟机先确认目标类已完成必要的加载与初始化，再为对象分配空间并初始化默认值，最后执行构造过程。HotSpot 常通过线程本地分配缓冲区减少并发分配竞争，但这不是 JVM 规范要求。

观察堆配置SHELL
java -Xms512m -Xmx512m \
  -Xlog:gc*:file=gc.log:time,uptime,level,tags \
  -jar archive-app.jar

jcmd <pid> GC.heap_info
jcmd <pid> GC.class_histogram

固定初始与最大堆可减少运行中扩容变化，但是否适合生产仍取决于容器限制、业务峰值和垃圾收集器。

05

REACHABILITY

对象何时不再存活

垃圾收集器通常从一组 GC Roots 出发遍历对象图。无法再从根集合到达的对象，才具备被回收的条件。静态字段、活动线程栈中的引用、JNI 引用等都可能让对象保持可达。

  • 01强引用存在时，对象通常不会被回收
  • 02软、弱、虚引用具有不同的可达性语义
  • 03ThreadLocal 的值可能因线程长期存活而滞留
  • 04缓存若没有边界，也会形成持续可达的对象图
资源释放不等于对象回收

文件、Socket 和数据库连接应使用 try-with-resources 或明确生命周期关闭，不能等待 GC 替代业务资源管理。

06

COLLECTION

回收、晋升与代际假设

很多收集器利用“多数对象朝生夕灭”的经验进行分代管理，新对象通常先进入年轻代，长期存活的对象可能晋升到老年代。但代际划分属于收集器与虚拟机实现，不是 JVM 规范对堆的统一要求。

Minor / Young GC

主要处理年轻代，频率与对象分配速率密切相关。

allocation pressure
Old / Mixed Collection

处理老年代或混合区域，行为依赖所选收集器。

collector dependent
Full GC

通常影响范围更大，但术语和触发原因也依赖实现，应结合 GC 日志判断。

read the log
07

NATIVE MEMORY

堆外还有哪些内存

  • ✦Metaspace 使用本地内存保存 HotSpot 类元数据
  • ✦DirectByteBuffer 背后可能分配直接内存
  • ✦每个线程需要本地线程栈与运行时结构
  • ✦JIT 编译代码、GC 数据结构和本地库也会占用内存
Native Memory TrackingSHELL
# NMT 必须在 JVM 启动时开启，会带来额外开销
java -XX:NativeMemoryTracking=summary -jar archive-app.jar

jcmd <pid> VM.native_memory summary

容器 OOM 但 Java 堆未满时，应把线程数、直接内存、Metaspace 和本地库纳入排查，而不是只继续增大 -Xmx。

GRAPH MODEL

顶点和边必须具有明确业务语义

无向图

关系双向成立，如物理连接或互为好友。

u — v
有向图

关系只沿指定方向成立，如依赖与关注。

u → v
带权图

边携带距离、时间、价格或风险。

weight
多重图

同一对顶点间允许多条业务不同的边。

parallel edges

先定义自环、重复边、孤立点、不可达和权重单位，再决定去重与存储规则。

02

REPRESENTATION

邻接表服务稀疏图，矩阵服务稠密关系

  • ✦邻接表空间通常为 O(V + E)，适合枚举某顶点的相邻边
  • ✦邻接矩阵空间为 O(V²)，判断两点是否直接相连简单
  • ✦边对象需要明确终点、权重和业务标识
  • ✦外部字符串 ID 应映射为紧凑内部编号，降低内存开销
  • ✦超大图要考虑分片、外存和流式计算，不能全部装入单 JVM
03

BREADTH-FIRST SEARCH

BFS 按层推进，解决无权最短步数

BFS 使用队列，从起点开始先访问距离一条边的顶点，再访问距离两条边的顶点。对无权图或所有边权相同的图，首次到达顶点时即可确定最少边数。

广度优先搜索PSEUDOCODE
queue.offer(source); visited[source] = true
while queue not empty:
  v = queue.poll()
  for each w in adjacency[v]:
    if not visited[w]:
      visited[w] = true; edgeTo[w] = v; queue.offer(w)

标记访问应发生在入队时，避免同一顶点被多个前驱重复入队。时间复杂度在邻接表模型下为 O(V + E)。

04

DEPTH-FIRST SEARCH

DFS 沿一条路径深入再回溯

DFS 适合连通分量、可达性、环检测、拓扑排序和回溯搜索。递归实现简洁，但图深不可控时可能耗尽线程栈，应改用显式栈。

  • 01无向图环检测要排除立即返回父节点的边
  • 02有向图常用未访问、访问中、已完成三种状态识别回边
  • 03拓扑排序只对有向无环图成立
  • 04遍历所有分量时要从每个未访问顶点重新启动
05

SHORTEST PATH

Dijkstra 只适用于非负权边

Dijkstra 每次从优先队列取出当前距离最小的顶点，并用它松弛相邻边。非负权保证已确定的最短距离不会被之后路径推翻。

优先队列松弛PSEUDOCODE
dist[source] = 0; queue.offer(source, 0)
while queue not empty:
  (v, d) = queue.poll()
  if d != dist[v]: continue
  for each edge(v, w, weight):
    if dist[v] + weight < dist[w]:
      update dist[w], edgeTo[w], and queue
出现负权就停止套用

负权边需要 Bellman–Ford 等其他方法；负环存在时，最短路径可能没有有限解。

06

ENGINEERING CHECKLIST

图算法落地检查清单

  • ✦确认有向、无向、权重范围和是否允许负值
  • ✦距离使用 long 并在相加前处理溢出与不可达哨兵
  • ✦保存 edgeTo 才能还原路径，仅保存 dist 只能得到距离
  • ✦用空图、断连、重复边、自环和极大权重测试
  • ✦优先队列若不支持 decrease-key，可加入新条目并跳过过期值
  • ✦把 O(V + E) 或 O((V + E) log V) 与真实规模上限对照
